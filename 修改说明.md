## main.js 


> 是项目的入口文件，负责初始化项目的配置、路由、组件、工具函数、Vue实例等。

一般写法

```
const app = createApp(App)

const pinia = createPinia()
pinia.use(piniaPluginPersistedstate)

app.use(pinia)
app.use(router)
app.use(ElementPlus, { locale: zhCn })

for (const [key, component] of Object.entries(ElementPlusIconsVue)) {
  app.component(key, component)
}

initComponents(app)

app.mount('#app')
```

项目中一般会有一些全局组件、工具函数、插件等，需要在 main.js 中进行初始化。
```
;(async () => {
  // 项目相关配置 config.js
  const config = await initConfig()  

  // 抽离出的路由配置 router.js
  const routes = await initRoutesConfig() 

  // 日志库初始化(iframe中不打印日志), 日志打印统一使用log.info | error | trace | warn()
  iframeMessage.isIframe
    ? log.init({ ...config.logOption, closeAll: true })
    : log.init(config.logOption)
  window.log = log

  // 初始化项目的配置、路由、组件、工具函数、Vue实例等 boot.js
  await initBoot(config)

  const router = initRouter(routes)

  initUtils()

  initComponent()

  initVue(router, store)
})()
```

## 项目的样式是怎么设计

### 总体架构



- 全局使用 CSS 变量作为主题与尺寸基准，文件在 config-style.css ，定义颜色、字体层级、卡片/弹窗等可配置变量
- 组件局部样式统一基于一个 SCSS规则库，通过 @use '@/assets/scss/rules' as * 引入，集中提供布局、字体、颜色、主题切换等 mixin： rules.scss
- 对 Element-UI 做了系统化二次样式封装，按组件拆分 SCSS 并在入口聚合： index.scss
- 全局 reset 与基础公共样式分别在 public.css 和 globalPublic.scss

### 字体设置

设置分为两部分 ： 全局字体设置 or 字体图标设置


图标方案来自 iconfont.cn （阿里巴巴矢量图标库），准备好字体库


1. 定义图标字体库

```
@font-face {
    font-family: 'iconfont'; /* Project id 3146932 */
    src: url('../font/iconfont.woff2') format('woff2');
}

.iconfont {
    font-family: 'iconfont', sans-serif !important;
    font-size: calc(18 * 1rem / 16);
    line-height: 1;
    font-style: normal;
    -webkit-font-smoothing: antialiased;
    -moz-osx-font-smoothing: grayscale;
    cursor: pointer;
}

```
2. 使用
 ```
 <!-- 1. 实体编码方式 -->
<em class="iconfont">&#xe82c;</em>

<!-- 2. 动态绑定方式 -->
<em class="iconfont" v-html="menu.icon" />

<!-- 3. 配合其他类名 -->
<em class="iconfont biz-success">&#xe830;</em>
```




### 主题机制

- 根节点通过属性 srm-theme-model 控制明暗主题，值为 dark|light；同时支持 srm-care-model 控制“关怀模式”增大字号： useTheme.js 、 config-style.css:light区块 、 config-style.css:care区块
- SCSS 中通过 mixin whenTheme/lightTheme/darkTheme 按主题分支输出样式，所有颜色使用 colorValue(...) 从 CSS 变量读取： rules.scss:主题判断 、 rules.scss:colorValue
- 运行时主题切换由 hooks 管理：设置 --color、srm-theme-model、srm-care-model，并同步到 Vuex；提供动画与用户配置持久化： useTheme.js 、 store/theme
- UI 内置主题切换器，选择主题色与关怀模式： ThemeSwitcher.vue

### SCSS工具库（常用）

- 布局与尺寸：wh/whrem、flexCenter、tableWheelLayout 等，用来快速构建栅格与弹性布局： rules.scss
- 字体系统：fontStyle(level) 统一字号/行高/粗细，关怀模式自动提升字号： rules.scss:fontStyle 、 config-style.css:字体变量
- 颜色与主题：themeColor/successColor/errorColor/disabledColor 等 mixin；所有颜色基于 CSS 变量，支持过渡动画： rules.scss:颜色系列 、 rules.scss:baseTransition
- 组件化块：searchBarStyle、foregroundStyle 等抽象了通用页面模块样式；在页面直接 @include 使用： globalPublic.scss
Element-UI 定制

- 针对各个 Element-UI 组件建立主题一致的样式片段，如按钮/菜单/标签/加载等，并在 index.scss 聚合引入
- 示例：菜单悬停边框/背景使用主题色变量；标签在明亮主题下有不同的边框与背景： RdappElMenu.scss 、 RdappElTag.scss
使用方式

- 在组件 Scoped 样式中统一引入规则库后使用 mixin：
  - 引入：style lang="scss" + @use '@/assets/scss/rules' as *
  - 示例用法：搜索栏 InspectConfigCategory.vue:样式 、栅格 inspectionGeneralConfig/Index.vue
- 主题变量在运行时通过 hooks 更新，组件中只需使用 var(--...) 或 colorValue(...)，无需关心切换细节
### 设计原则

- 动态主题与语义色：统一从 CSS 变量取色，确保主题切换时不需要改样式代码
- 组件化 mixin：布局、字体、模块样式用 mixin 抽象，保持页面样式稳定一致
- 响应式与可达性：pxToRem 保持尺寸与可读性，关怀模式通过根属性提升一组字体变量
- 与库风格统一：对 Element-UI 做一致的主题化封装，避免组件风格不一致
